# コードレビューチェックリスト

レビュー時に体系的に確認するためのチェックリストです。

## コード品質

- [ ] 一貫したスタイルと慣習に従っている
- [ ] 名前が説明的で命名規則に従っている
- [ ] 関数/メソッドが小さく焦点を絞っている
- [ ] コードの重複がない
- [ ] 複雑なロジックがシンプルな部分に分割されている
- [ ] エラー処理が適切
- [ ] コメントアウトされたコードやチケットのない TODO がない

## セキュリティ

- [ ] コードやログに機密データがない
- [ ] すべてのユーザー入力に入力検証がある
- [ ] SQL インジェクション脆弱性がない
- [ ] 認証と認可が適切に実装されている
- [ ] 依存関係が最新でセキュア

## テスト

- [ ] 新しいコードに適切なテストカバレッジがある
- [ ] テストの名前が適切で焦点を絞っている
- [ ] テストがエッジケースとエラーシナリオをカバー
- [ ] テストが独立して決定論的
- [ ] 常にパスするテストやコメントアウトされたテストがない

## パフォーマンス

- [ ] 明らかなパフォーマンス問題がない（N+1、メモリリーク）
- [ ] キャッシュの適切な使用
- [ ] 効率的なアルゴリズムとデータ構造
- [ ] 適切なリソースクリーンアップ

## アーキテクチャ

- [ ] 確立されたパターンと慣習に従っている
- [ ] 適切な関心の分離
- [ ] アーキテクチャ違反がない
- [ ] 依存関係が正しい方向に流れている

## ドキュメント

- [ ] パブリック API がドキュメント化されている
- [ ] 複雑なロジックに説明的なコメントがある
- [ ] 必要に応じて README が更新されている
- [ ] 破壊的変更がドキュメント化されている
